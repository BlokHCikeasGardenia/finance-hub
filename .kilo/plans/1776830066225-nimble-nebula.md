# Plan: Period Filter Feature for Hunian Administration

## Overview
Add a period dropdown filter to the Hunian Administration page (`hunian-administrasi.js`) with two options:
- **"6 Periode Terakhir"** (default) - shows bills from the 6 most recent billing periods that **actually have bills** (any type), ordered by period date descending. This can show up to 12 items (6 IPL + 6 Air) if both types exist for each period.
- **"Semua Periode"** - shows complete billing history (all periods).
Each hunian card will have a **"Selengkapnya"** button that opens a modal popup with the full payment details (all periods).

## User Choices Confirmed
- **Filter Options**: "6 Periode Terakhir" (default) + "Semua Periode"
- **Detail View**: Modal popup (stays on same page)
- **Filtering**: Client-side (fetch all bills, then filter by period IDs that have data) — **CHANGED** from server-side to handle prepaid future periods correctly
- **Limit Definition**: The 6 most recent periods (by `tanggal_awal` DESC) that have at least one bill (IPL or Air). If a period has no bills, it's skipped. This handles prepaid future periods correctly.

## Why Client-Side Filtering?
**Scenario:** Residents prepay IPL through December 2026. Period records exist for Apr-Dec 2026, but only some have bills.
- **Server-side** (original): Fetch top 6 period records by date → may include empty periods → shows no data
- **Client-side** (new): Fetch ALL bills (already done), extract periods that have bills, pick 6 most recent → guarantees showing periods with actual data

**Trade-off:** Slightly more data transferred, but simpler logic and correct behavior with prepayments. Bills per hunian are limited (~12-24) so performance is fine.

### 4. Client-Side Period Filtering Logic
After fetching bills in `renderHunianAdminCards()` (line ~260-311):
1. Collect all unique `periode_id` from the fetched `ipl` and `air` bills
2. Fetch period details (`nama_periode`, `tanggal_awal`) for those IDs (batch query)
3. Sort periods by `tanggal_awal` DESC
4. If `periodFilter === '6periode'`: take first 6 period IDs
5. Filter the bills array to only those belonging to selected period IDs
6. Sort filtered bills by period using existing `sortTagihanByPeriode()`

```javascript
// After fetching ipl and air bills:
const allBills = [...iplDetails, ...airDetails];

// Get unique period IDs from bills
const periodIdsWithBills = [...new Set(allBills.map(b => b.periode_id).filter(id => id))];

// Fetch period details for these IDs
const { data: periodData } = await supabase
    .from('periode')
    .select('id, nama_periode, tanggal_awal')
    .in('id', periodIdsWithBills);

// Sort periods by tanggal_awal DESC
const sortedPeriods = (periodData || []).sort((a, b) => 
    new Date(b.tanggal_awal) - new Date(a.tanggal_awal)
);

// Determine which periods to show
const selectedPeriodIds = periodFilter === '6periode' 
    ? sortedPeriods.slice(0, 6).map(p => p.id)
    : sortedPeriods.map(p => p.id);

// Filter bills to only selected periods
const filteredBills = allBills.filter(bill => selectedPeriodIds.includes(bill.periode_id));

// Sort and assign to hunian
const semuaTagihan = sortTagihanByPeriode(filteredBills);
```

## Edge Cases
- Fewer than 6 periods with bills: show all available
- Periods with no bills: automatically excluded from "6 Periode Terakhir"
- Prepaid future periods: included if they have bills
- Empty hunian (no bills at all): modal shows empty state
